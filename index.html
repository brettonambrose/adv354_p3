LAMP
 
What is a LAMP Stack?

LAMP is an acronym that stands for Linux, Apache, MySQL, and PHP, but there are many derivatives. The combination of these technologies creates a bundle known as a "stack." Commonly, LAMP stacks are made-up of open-source technologies. However, there are variations such as WAMP which uses Windows and MAMP on MacOS. The modularity of the stack has become popular due to its support for content management systems like WordPress. LAMP stacks provide reliability and uptime due to their modular nature making them essential for large websites with high demand. System administrators often deploy LAMP stacks due to their load-balancing features.

What do the parts of the stack do?
Linux

The operating system sits at the core of the LAMP stack. Traditionally, this is Linux (an operating system developed by Linus Torvalds in 1991.) The operating system interprets commands issued to it by other parts of the LAMP stack and allocates resources to services.

Apache

In a traditional LAMP stack, Apache is the webserver. The webserver does the work of sending HTML to your computer that is then displayed by your brower. Apache is robust, extensible, and open-source. Countless websites have relied on it for years.

MySQL

MySQL is a relational database server. In a LAMP stack, this component is responsible for storing data like user lists, passwords, or eCommerce inventory data. By using structured query language (SQL), developers have virtually unlimited ways to manipulate data in database tables.

PHP

Using a recursive acronym, the PHP Hypertext Processor is a general purpose scripting language that allows the webserver to interact directly with the database server. Its uses and extensibility are limitless. PHP is at the core of most interactive websites.
